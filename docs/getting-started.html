Quick start

Get Grain on the page in minutes.

You can use Grain as plain old CSS and JavaScript, or import it once from npm and let the browser do the rest. The semantic HTML you already know is the public API.

1. Install

Use the CDN snippet for prototypes or the npm package for apps and design systems.

CDN 
<link rel="stylesheet" href="https://unpkg.com/grain-ui/dist/grain.min.css">
<script src="https://unpkg.com/grain-ui/dist/grain.min.js" defer></script>
npm 
npm install grain-ui

import "grain-ui/css";
import "grain-ui";

2. Your first page

Start with native elements. Grain styles them semantically, so your markup stays clean and portable.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <link rel="stylesheet" href="https://unpkg.com/grain-ui/dist/grain.min.css" />
    <script src="https://unpkg.com/grain-ui/dist/grain.min.js" defer></script>
    <title>My Grain page</title>
  </head>
  <body>
    <main style="max-width: 48rem; margin: 0 auto; padding: 2rem;">
      <h1>Hello, Grain</h1>
      <p>This page uses only semantic HTML.</p>
      <button>Primary action</button>
    </main>
  </body>
</html>

3. Add a button, a form, and a modal

Variants and validation are driven by attributes, not component classes.

A valid email is required.
Ship this change?

The modal uses the native <dialog> element and closes on backdrop click or Esc.

 
<label for="email">Email</label>
<input id="email" type="email" aria-invalid="true" />
<small role="alert">A valid email is required.</small>

<button data-modal="demo-modal">Open modal</button>
<grain-modal id="demo-modal">
  <dialog>
    <header>
      <strong>Confirm action</strong>
      <button data-close data-variant="ghost">✕</button>
    </header>
    <section>...</section>
    <footer>
      <button data-close data-variant="outline">Cancel</button>
      <button>Confirm</button>
    </footer>
  </dialog>
</grain-modal>

4. Customize colors

Most themes start by changing the hue, accent, and surface tokens. You can override them globally or on a subtree.

:root {
  --gr-hue: 35;
  --gr-accent: hsl(35 86% 46%);
  --gr-accent-hover: hsl(35 86% 40%);
  --gr-accent-light: hsl(35 88% 93%);
  --gr-radius: 12px;
}

Tip: use src/tokens.css as the single source of truth. That file is the contract between Grain and your theme.

5. Use the Web Components when you need behavior

Tabs, toasts, dropdowns, and modal helpers are custom elements with shadow DOM turned off, so they inherit your global styles naturally.

Tabs are progressively enhanced and keyboard navigable.

Use arrow keys, Home, End, Enter, or Space.

 
<grain-tabs>
  <div role="tablist">
    <button role="tab">Overview</button>
    <button role="tab">Usage</button>
  </div>
  <div role="tabpanel">...</div>
  <div role="tabpanel">...</div>
</grain-tabs>

<button onclick="GrainToast.show('Saved successfully.')">Show toast</button>

6. Framework integration notes

Grain works best when you let the browser keep doing browser things.

React

Import the CSS once, import grain-ui for side effects, and use native props like aria-invalid directly.

Vue / Nuxt

Register the stylesheet globally. Custom elements work out of the box in Vue 3, especially when you keep them as native tags.

Svelte / Astro

Grain fits naturally because the markup stays close to the platform. Import once in your layout and use semantic HTML in components.
