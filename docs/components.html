Component library

Semantic components, documented as HTML.

Each section below includes a live preview, a markup example, and the attributes or ARIA roles that control behavior. Grain tries to stay close to the platform rather than inventing a new API.

Button

<button>Primary</button>
<button data-variant="outline">Outline</button>
<button data-variant="danger">Delete</button>
<button aria-busy="true">Saving</button>
AttributeMeaning
data-variantVisual style: outline, ghost, danger, subtle.
data-sizeUse sm or lg for compact or roomy buttons.
aria-busy="true"Adds a spinner and communicates loading state.

Form controls

Please enter a valid email. 
<label for="email">Email</label>
<input id="email" type="email" aria-invalid="true" />
<small role="alert">Please enter a valid email.</small>
AttributeMeaning
aria-invalid="true"Applies the error border treatment without a separate class.
disabledDims controls and removes pointer interactions.
role="group"Use on wrappers when related fields belong together outside a <fieldset>.

Tabs

Tabs are enhanced by the <grain-tabs> custom element.

Use arrow keys or Home/End to move between tabs.

Panels get the right aria-labelledby and hidden state automatically.

 
<grain-tabs>
  <div role="tablist">
    <button role="tab">Summary</button>
    <button role="tab">Usage</button>
  </div>
  <div role="tabpanel">...</div>
  <div role="tabpanel">...</div>
</grain-tabs>
Key / attributeBehavior
role="tablist"Defines the tab group container.
role="tab"Receives aria-selected and managed tabindex.
← → Home EndKeyboard navigation between tabs.

Modal

Deploy now?

This helper wires trigger buttons to a native dialog, keeps backdrop click behavior simple, and respects browser focus handling.

 
<button data-modal="billing-modal">Open modal</button>
<grain-modal id="billing-modal">
  <dialog>...</dialog>
</grain-modal>
Key / attributeBehavior
data-modal="id"Associates a trigger with a matching <grain-modal id="...">.
data-closeCloses the dialog when applied to a button inside the modal.
EscHandled by the native dialog element to close the modal.

Actions

A card with a dropdown menu in its header — the most common pattern.

Billing
Export

Dropdown items can be any focusable element inside [role="menu"].

<grain-dropdown>
  <details>
    <summary>Actions</summary>
    <div role="menu">
      <button type="button">Rename</button>
      <button type="button">Duplicate</button>
    </div>
  </details>
</grain-dropdown>
Key / attributeBehavior
role="menu"Identifies the menu container and enables keyboard targeting.
↓ ↑Open the menu and move between items.
EscCloses the menu and returns focus to the summary trigger.
