Semantic UI without the framework tax.

Grain styles the HTML you already write, adds tiny browser-native behavior where it helps, and stays fully themeable through plain CSS variables.

Quick start

Two distributable files, semantic primitives, and tiny custom elements for the interactive pieces.

npm install @rachanpadmanabha/grain-ui

import "@rachanpadmanabha/grain-ui/css";
import "@rachanpadmanabha/grain-ui";
  • Native HTML elements are the public API
  • Theme everything from :root tokens
  • Works in plain HTML, React, Vue, Astro, and Svelte
01

Semantic-first

A <button> looks right. A <dialog> works. Attributes like aria-invalid and data-variant drive state. No class soup.

02

Tiny by design

Grain ships as one CSS file and one JS file. You can use the whole library with a pair of tags and no build step at all.

03

Zero dependencies

No React lock-in, no utility compiler, no preprocessors. Include it from a CDN or install it from npm and go.

04

Themeable with tokens

Override a few CSS variables or replace every token. The library is designed to be rethemed from :root.

05

Dark mode built in

Grain respects prefers-color-scheme automatically and also supports explicit data-theme overrides.

06

Framework-friendly

Custom elements for tabs, modals, dropdowns, and toasts work the same way in plain HTML, React, Vue, Astro, and Svelte.

Theming

One hue variable, infinite themes.

The default palette derives from --gr-hue. Shift the system in place, then layer more exact token overrides when you need them.

Override radius, fonts, surfaces, focus rings, and dark mode tokens after the hue if you need a more branded result.
